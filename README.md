# CS210-Project-One

Summarize the project and what problem it was solving.
  This project was to create a code wherein a time could be input, and the program would output that time in both 12 and 24 hour clock format, with the ability to change second, hour, and minute.
  
What did you do particularly well?
  Looking back through my code, I feel particularly happy with my inline comments, as even several weeks later, I am able to follow my past logic quickly and succicently through reading my comments.  The code follows a clear consisent organizitation that also keeps it easy to read.
  
Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
  Given the oppurtunity to redo project one now, I think including a header file for declaring functions would further improve readability and maintanability of the program.  I also found that I did not have an addition for my time functions, allowing them to tick up one if the needed (for example if a second was added to already 59 seconds, it reset to 0, but the minute did not tick up one), and would need to be improved.
  
Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
  Finding how to make the 24 hour convert to the 12 hour I found to be challenging, as I really needed to think objectively about the difference between the two clocks in a way I hadn't really had to consider, for example that they would read as the same time for the first 12 hours but not the second half, my brain did not initially think through that thought process, and I really needed to tackle the problem with a rational brain looking through the lens of programming. 
  
What skills from this project will be particularly transferable to other projects or course work?
  This project gave me a great foundation for laying down C++ code successfully, and something I think is particularly transferable was thinking about the user functioning of this project throughout the code, remembering to place 'invalid' buffers for wrong entries, which without could completely crash a code. 
  
How did you make this program maintainable, readable, and adaptable?
  My inline comments create a very nicely readable code, and keeping consistent spacing and indentation also keeps the coding looking really clean.
